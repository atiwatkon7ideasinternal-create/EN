# 🎯 เกมฝึกจำสระภาษาอังกฤษ (English Vowel Practice Game)

> เอกสารออกแบบ (Spec) สำหรับสร้างเว็บเกมฝึกจำเสียงสระภาษาอังกฤษ
> เทียบกับสระไทย — เขียนเพื่อให้นำไปสร้างเว็บได้จริง

---

## 1. ภาพรวม (Overview)

เว็บเกมสำหรับ **ฝึกจำสระภาษาอังกฤษ** โดยเฉพาะ "สระผสม" (vowel combinations / digraphs)
ผู้เล่นจะเห็นสระผสมภาษาอังกฤษทีละตัว แล้ว **ทายว่าตรงกับเสียงสระไทยตัวไหน**
เล่นวนจนครบทุกตัว จากนั้นสรุปผลเป็น **คะแนนรวม + เปอร์เซ็นต์ความถูกต้อง + จำนวนครั้งที่ทายถูก**

### เป้าหมาย
- ช่วยจำว่าสระผสมอังกฤษออกเสียงแบบไหน (เทียบกับสระไทยที่คุ้นเคย)
- เล่นได้เรื่อย ๆ ซ้ำ ๆ จนจำได้แม่น
- เห็นความก้าวหน้าของตัวเอง (สถิติคะแนน)

### กลุ่มผู้ใช้
- คนไทยที่อยากจำสระภาษาอังกฤษ (เริ่มต้น – กลาง)
- เล่นคนเดียว ไม่ต้องล็อกอิน

## 2. รูปแบบการเล่น (Game Flow)

```
[หน้าเริ่มต้น] → [เริ่มเกม] → [คำถามที่ 1] → [คำถามที่ 2] → ... → [คำถามสุดท้าย] → [หน้าสรุปผล]
                                   ↑___________ วนจนครบทุกสระ ___________↑
```

1. **หน้าเริ่มต้น (Start Screen)**
   - ชื่อเกม + ปุ่ม "เริ่มเล่น"
   - เลือกโหมด (ดูข้อ 7) — มีหรือไม่มีก็ได้ใน v1

2. **หน้าเล่นเกม (Quiz Screen)** — วนซ้ำต่อคำถาม
   - แสดงสระผสมอังกฤษตัวใหญ่ ๆ เช่น **`ee`**
   - แสดงตัวเลือกสระไทย 4 ตัว (1 ถูก + 3 ตัวลวง) แบบสุ่มลำดับ
   - ผู้เล่นกดเลือก → เฉลยทันที (ถูก = เขียว, ผิด = แดง + ไฮไลต์ข้อที่ถูก)
   - แสดงคำตัวอย่าง (example word) เพื่อช่วยจำ เช่น `see`, `tree`
   - กดปุ่ม "ถัดไป" เพื่อไปข้อต่อไป

3. **หน้าสรุปผล (Result Screen)**
   - คะแนนรวม เช่น `8 / 10`
   - เปอร์เซ็นต์ความถูกต้อง เช่น `80%`
   - จำนวนครั้งที่ทายถูก / ทั้งหมด
   - รายการสระที่ตอบผิด (เพื่อกลับไปทบทวน)
   - ปุ่ม "เล่นอีกครั้ง" และ "เล่นเฉพาะข้อที่ผิด"

## 3. ข้อมูลสระ (Vowel Data)

ตารางข้อมูลหลักของเกม — เป็นหัวใจของเว็บ เก็บเป็น array of objects ใน JS/JSON
แต่ละตัวมี **คำตัวอย่าง + ประโยคตัวอย่าง** เพื่อช่วยให้จำเสียงสระได้ง่ายจริง ๆ

รวมทั้งหมด **42 สระผสม** แบ่งเป็น **9 หมวดย่อยตามตระกูลเสียง** (เล่นทีละหมวดจะสั้น เล่นจบไว)
แต่ละตัวมีคำตัวอย่าง + ประโยคตัวอย่าง + เคล็ดจำ

## 3.11 หลักการออกแบบให้ "จำได้ง่ายจริง" (Memory-Aid Design)

หัวใจที่ทำให้เกมนี้ช่วยจำได้จริง ไม่ใช่แค่ทายถูก/ผิด:

1. **เห็นเสียง + เห็นตัวอย่างพร้อมกัน** — ทุกข้อโชว์คำ + ประโยคที่ไฮไลต์สระเป้าหมาย
   ตัวอย่าง: `a-e` → **g<u>a</u>me**, *"I play a **game**."* → สมองผูกเสียงกับบริบท
2. **ฟังเสียงได้ 🔊** — กดฟังคำ/ประโยค (Web Speech API) จำเสียงจริง ไม่ใช่แค่ตัวอักษร
3. **จัดกลุ่มตามเสียง** — สระเสียงเดียวกันอยู่ด้วยกัน (`ai`=`ay`=`a-e` → เอ) เห็นแพทเทิร์น
4. **เคล็ดช่วยจำ (hint)** — แต่ละสระมีประโยคจำสั้น ๆ เช่น *"e เงียบ แต่มีเวทมนตร์"*
5. **ทบทวนข้อที่ผิดบ่อยขึ้น (Spaced Repetition)** — ข้อที่ตอบผิดจะถูกหยิบมาถามซ้ำถี่กว่า
6. **เล่นสั้น ๆ แต่บ่อย** — 1 รอบไม่เกิน 1–2 นาที เล่นซ้ำได้เรื่อย ๆ จนจำแม่น

## 3.12 งานวิจัยรองรับการออกแบบ (Research Evidence & Citations)

หลักการทุกข้อในเกมนี้อ้างอิงงานวิจัยด้าน cognitive science / second-language acquisition
ที่ผ่านการตรวจสอบ (peer-reviewed meta-analyses เป็นหลัก) — สรุปพร้อม effect size (Hedges' *g*)
และวิธีที่นำมาใช้จริงในแอป

> *g* = effect size: ~0.2 เล็ก, ~0.5 กลาง, ~0.8+ ใหญ่ (ยิ่งสูงยิ่งได้ผล)

| # | หลักการ | หลักฐาน (effect size) | นำมาใช้ในแอปอย่างไร |
|---|---------|----------------------|---------------------|
| 1 | **Retrieval practice / Testing effect** — การ "ดึงความจำ" จำได้ดีกว่าการอ่านทบทวน | *g* ≈ **0.51** เทียบกับทบทวนซ้ำ, **0.93** เทียบกับดูเฉย ๆ [1][2] | ทั้งเกมเป็นควิซ ผู้เล่นต้องตอบทุกข้อ ไม่ใช่แค่ดูการ์ด |
| 2 | **Production/Generation effect** — ผลิตคำตอบเอง (นึก/พิมพ์) ดีกว่าเลือกจากตัวเลือก | recall *g* ≈ **0.61–0.81** vs recognition *g* ≈ **0.29–0.36** [2] | โหมด "พิมพ์เอง" (ยากขึ้น) ในการสะกดคำ |
| 3 | **Corrective feedback** — เฉลยทุกครั้งหลังตอบช่วยเสริมผลอย่างมาก | มี feedback *g* ≈ **0.73** vs ไม่มี **0.39** [2][1] | เฉลยถูก/ผิดทันที + โชว์คำ ประโยค เคล็ดจำ และฟังเสียงซ้ำ |
| 4 | **Spaced > massed** — กระจายการทบทวนข้ามเวลา (สำคัญสุดเรื่องตาราง) | *g* ≈ **0.74** [3] | Leitner box เก็บข้ามรอบ/ข้ามวัน (localStorage) ตัวที่ผิดถูกถามถี่กว่า |
| 5 | **Expanding intervals** — ขยายช่วงห่างทีละน้อย โดยรวมไม่ต่างจากช่วงคงที่ แต่ช่วยเมื่อทบทวนหลายรอบ | expanding vs uniform *g* ≈ **0.034** (n.s.) แต่ดีขึ้นตามจำนวนครั้งที่ทบทวน [3][4][5] | ถามซ้ำในรอบแบบขยายช่วง **2 → 5 → 9 ข้อ** (ทำพอประมาณ ไม่ over-engineer) |
| 6 | **HVPT (High-Variability Phonetic Training)** — ฟังหลายคนพูดในบริบทหลากหลาย + feedback ราย trial | *g* ≈ **0.92** (สระ **1.00**!) คงทน decay ≈ **−0.08** ใน ~2.3 เดือน [6] | สุ่มเสียงผู้พูด (talkers) หลายคน + สุ่มคำตัวอย่างต่างกันทุกครั้ง |
| 7 | **Identification + text label** — แมปเสียง→สัญลักษณ์ตัวอักษร ดีกว่าแบบ same/different และดีกว่ารูปภาพ | identification *g* ≈ **0.95** vs discrimination **0.57**; label ตัวอักษร **0.90–1.03** vs รูปภาพ **0.47** [6] | ทายโดยแมปเสียง → "สระไทย"/ตัวสะกด (เป็น text ล้วน ไม่ใช้รูป) |
| 8 | **Interleaving (ใช้อย่างระวัง)** — สลับหมวดช่วยเมื่อหมวดสับสนกัน แต่เคยให้ผลลบกับ "คำ" | โดยรวม *g* ≈ **0.42** แต่กับ word materials *g* ≈ **−0.39** [7] | สลับกลุ่มสระได้ **แต่ให้เลือกเล่นทีละกลุ่ม (blocking) ได้ด้วย** |
| 9 | **สอนกฎครั้งเดียวไม่พอ** — อธิบาย grapheme–phoneme รอบเดียวไม่ลดการรบกวนจากการสะกด | ไม่มีผลบวกอย่างมีนัยสำคัญ [8] | "เคล็ดจำ" เป็นแค่ตัวเสริม — แกนหลักคือฝึกดึงความจำซ้ำ ๆ พร้อมเสียง |

### ⚠️ ข้อที่ "ถูกหักล้าง" (อย่านำมาออกแบบ)
จากการตรวจสอบแบบ adversarial (ต้องผ่าน 3 ผู้ตรวจ) มี 3 ข้อที่ **ไม่ผ่าน** — จึงไม่ยึดเป็นหลัก:
- ❌ การฝึกสระที่ "สะกดทับซ้อนกัน" ให้ผลจำระยะยาวดีกว่าแบบแยกชุด (โหวต 0–3)
- ❌ ความทับซ้อนสูง = desirable difficulty ที่ได้ผลชัด (โหวต 1–2)
- ❌ การ "เลื่อนการดึงความจำครั้งแรกให้ช้าลง" คือปัจจัยหลักของการจำระยะยาว (โหวต 0–3)

> หมายเหตุ: ตัวลวงที่ "สับสนง่าย" ในเกม (เช่น `ai`/`ay`/`a-e` → เอ) อิงหลัก **identification ของ HVPT [6]**
> (ฝึกแยกแยะเสียง/การสะกดที่แมปเสียงเดียวกัน) — คนละเรื่องกับข้อที่ถูกหักล้างด้านบน

## 3.13 โหมดเล่น (Game Modes)

เกมมีหลายโหมดเพื่อฝึกจำจากหลายมุม:

### 🅰️ โหมด 1: ทายสระ (Multiple Choice) — *โหมดหลัก v1*
- โชว์สระอังกฤษ → เลือกสระไทยที่ตรง (4 ตัวเลือก)
- เหมาะกับ "เริ่มต้นจำ"

### ✍️ โหมด 2: ฝึกสะกดคำ (Spelling) — *เพิ่มใหม่*
> ฝึกเขียนคำให้ถูกโดยเน้นที่ "สระ" ที่หายไป — ช่วยจำว่าสระเสียงนี้สะกดยังไง

**รูปแบบ 2A — เติมสระที่หายไป (Fill the vowel):**
```
ฟังเสียง 🔊 "game"  →  แสดง:  g [ __ ]  me
ผู้เล่นเลือก/พิมพ์สระที่หาย:  a-e  (หรือเลือกจากปุ่ม a-e / ai / ay)
เฉลย: g[a]me ✓
```

**รูปแบบ 2B — พิมพ์ทั้งคำ (Type the word):**
```
ฟังเสียง 🔊 + เห็นสระไทยใบ้ "เอ"  →  ช่องพิมพ์: [ ______ ]
ผู้เล่นพิมพ์:  game
ตรวจ: ถ้าสะกดถูก ✓  ผิดตรงไหนไฮไลต์ตัวที่ผิด (เทียบทีละตัวอักษร)
```

- ให้คะแนนเหมือนโหมดทาย (ถูก +1, เก็บคำที่ผิดไว้ทบทวน)
- เน้นสะกด "สระ" ถูกเป็นหลัก (พยัญชนะอาจใบ้ให้ก่อนได้ในระดับง่าย)
- ระดับความยาก: ง่าย = เติมเฉพาะสระ / ยาก = พิมพ์ทั้งคำเอง

### 🔁 โหมด 3: กลับด้าน (Reverse) — *อนาคต*
- โชว์สระไทย "เอ" → ทายว่าตรงกับสระอังกฤษตัวไหน (เลือกหลายตัวที่ถูก เช่น ai/ay/a-e)

## 4. ตรรกะเกม (Game Logic)

### การสุ่มคำถาม
- สุ่มลำดับสระทั้งหมดใหม่ทุกครั้งที่เริ่มเกม (shuffle)
- เล่นจนครบทุกตัวใน 1 รอบ (ไม่ถามซ้ำในรอบเดียว)

### การสร้างตัวเลือก (Choices)
- ตัวเลือกที่ถูก = `thai` ของข้อนั้น
- ตัวลวง 3 ตัว = สุ่มจาก `thai` ของสระอื่น (ห้ามซ้ำกับข้อที่ถูก และห้ามซ้ำกันเอง)
- สับลำดับตัวเลือกทั้ง 4 (shuffle)

### การให้คะแนน
```
ตอบถูก  → +1 คะแนน, นับ correctCount +1
ตอบผิด  → +0 คะแนน, เก็บข้อนี้ไว้ใน wrongList
```

### สรุปผล (ตอนจบ)
```
totalQuestions = จำนวนสระทั้งหมดในรอบ
correctCount   = จำนวนที่ตอบถูก
percentage     = Math.round((correctCount / totalQuestions) * 100)
```
แสดง: `correctCount / totalQuestions` และ `percentage%`
